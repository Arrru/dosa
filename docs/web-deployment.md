# 웹 배포 가이드

이 문서는 현재 Godot MVP 프로젝트를 브라우저에서 실행 가능한 형태로 Export하고, 도메인에 연결해 배포하는 절차를 정리한다.

## 1. 사전 조건
- Godot 4.6 이상 설치
- 현재 프로젝트를 로컬 PC에서 정상적으로 열 수 있어야 함
- 가능하면 HTTPS를 제공하는 정적 호스팅 사용

## 2. Godot에서 Web Export Preset 만들기
1. Godot 에디터에서 프로젝트 열기
2. 상단 메뉴 `Project > Export...`
3. `Add...` 클릭
4. `Web` 선택
5. Export 이름은 예: `web`
6. 출력 파일명은 `index.html` 로 설정

### 권장 설정
- Export 방식: 기본 Web Export
- Thread 사용: 특별한 이유가 없으면 단일 스레드 유지
- Canvas 크기/리사이즈는 기본 브라우저 적응 방식 유지 권장
- PWA는 필요할 때만 활성화

## 3. Export 시 생성되는 파일
일반적으로 아래 파일들이 함께 생성된다.

- `index.html`
- `index.js`
- `index.wasm`
- `index.pck`
- 기타 splash/asset 관련 부속 파일

이 파일들은 **같은 폴더에 함께 업로드**해야 한다.

## 4. 호스팅 후보
정적 파일 업로드가 가능한 호스팅이면 대부분 가능하다.

- Netlify
- Vercel
- Cloudflare Pages
- GitHub Pages
- Nginx / Apache 직접 서버

## 5. 도메인 연결
예시 흐름:
1. Export 결과물을 호스팅 서비스에 업로드
2. 서비스가 제공하는 기본 URL에서 먼저 실행 확인
3. 커스텀 도메인 연결
4. DNS에서 CNAME 또는 A 레코드 설정
5. HTTPS 인증서 적용 확인

## 6. 서버/호스팅 시 주의할 점
- `.wasm` 파일이 정상 제공되어야 함
- 가능하면 gzip/Brotli 압축 사용
- HTTPS 사용 권장
- iframe 삽입 환경에서는 브라우저 저장소 제한이 더 심해질 수 있음

## 7. 저장 시스템 주의 사항
현재 프로젝트는 `user://slot_*.json` 저장 구조를 사용한다.

웹 빌드에서는 이 저장이 브라우저 저장소를 사용하게 되므로 다음 상황에서 세이브가 유지되지 않을 수 있다.

- 시크릿/사생활 보호 모드
- 브라우저 저장공간 차단
- 사이트 데이터 삭제
- 일부 iframe/임베드 환경

따라서 정식 배포 전 반드시 실제 배포 URL에서 아래를 확인해야 한다.

1. 새 게임 시작
2. 저장
3. 페이지 새로고침
4. 이어하기 가능 여부 확인

## 8. 현재 프로젝트에서 이미 반영된 웹 대응
- 웹 배포를 고려해 렌더링 방식을 Compatibility로 조정
- 타이틀 설정 메뉴에서 웹 환경일 경우 창 크기 변경 대신 전체화면 요청 중심으로 처리
- 웹 저장소 제약 안내 메시지 추가
- 저장 실패 시 사용자 안내 문구 강화
- 웹 환경에서는 `종료` 버튼이 앱 종료 대신 안내 메시지를 표시

## 9. 배포 전 최종 체크리스트
- [ ] 로컬 Godot 에디터에서 Web Export 성공
- [ ] 로컬 또는 테스트 호스팅 URL에서 실행 확인
- [ ] 새 게임 / 이어하기 / 저장 / 새로고침 후 복구 확인
- [ ] 모바일 브라우저에서 레이아웃 확인
- [ ] 실제 도메인 연결 후 HTTPS 확인
- [ ] `.wasm` 로딩 에러가 없는지 브라우저 개발자 도구에서 확인
