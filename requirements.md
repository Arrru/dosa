# 괴담 도서관 MVP 프로토타입 요구사항 명세

## 목표
- Godot 4 기반의 포인트 앤 클릭 + 선택지 기반 크리처 미연시/괴담 도서관 게임의 플레이 가능한 MVP 프로토타입을 구현한다.
- 빈 저장소에서 바로 실행 가능한 프로젝트 구조와 핵심 루프를 만든다.
- 무료/교체 가능한 플레이스홀더 비주얼을 사용해 이후 실제 아트로 대체하기 쉬운 형태를 유지한다.

## 범위
### 이번 MVP에 포함
- 최초 실행용 인트로 흐름
- 이름 입력 및 재확인 후 확정
- 타이틀 메뉴
- 3개 세이브 슬롯 기반 새 게임 / 이어하기
- 도서관 단일 공간의 방향 전환형 탐색 화면
- 클릭 가능한 상호작용 지점
- 최대 3개 선택지 기반 대화
- 낮 → 밤 → 자유시간 반복의 단순 시간 루프
- 호감도 / 스킬 / 도감 / 매뉴얼 잠금 해제의 기본 구조
- 샘플 캐릭터 1명, 샘플 이벤트/대화 몇 개
- 세이브/로드 후 진행 상태 유지

### 이번 MVP에서 제외
- 다수 캐릭터와 풀 루트 콘텐츠
- 실제 완성 아트, 연출, 사운드 연출 고도화
- 복잡한 퍼즐, 전투, 미니게임 본격 구현
- 업적 전체, 엔딩 전체, 2회차 풀 메타 전개
- 로컬라이징, 외부 애드온 기반 에디터 확장

## 주요 기능
1. **인트로/이름 확정**
   - 영상 기반 인트로 또는 영상 미존재 시 대체 인트로
   - 플레이어 이름 입력
   - 재확인 후 확정, 확정 뒤에는 저장 데이터 기준으로 유지
2. **메인 메뉴**
   - 새 게임
   - 이어하기
   - 근무기록 오버레이 화면
   - 설정 오버레이 화면
   - 크레딧 오버레이 화면
3. **탐색 화면**
   - 북/동/서/남 방향 전환
   - 시점별 배경과 핫스팟 표시
4. **상호작용**
   - 책상, 금서 책장, 창문, 괴이 손님 자리, 서랍/침낭 등 샘플 상호작용
5. **대화/선택지**
   - 대사 표시
   - 최대 3개 선택지
   - 선택에 따른 호감도/스킬/도감/매뉴얼 변화
6. **진행 루프**
   - 행동 소모 시 시간대 전환
   - 낮/밤/자유시간 반복
7. **수집 및 성장 스텁**
   - 도감 해금
   - 매뉴얼 해금
   - 스킬 상승
   - 호감도 변화
   - 도감/매뉴얼은 목록+상세 보기 형태의 인게임 열람 UI 제공
8. **세이브 시스템**
   - 3 슬롯 저장/불러오기
   - 이름, 날짜, 시간대, 시점, 호감도, 스킬, 해금 상태 저장

## 제약 조건
- 외부 애드온 없이 Godot 기본 기능 중심으로 구성한다.
- 저장은 `user://` 기반 JSON 파일로 관리한다.
- 현재 저장소는 비어 있으므로 처음부터 프로젝트를 구성한다.
- 이미지 의존성이 있는 부분은 직접 만든 플레이스홀더 SVG로 대체 가능해야 한다.
- 실제 풀 콘텐츠가 아닌 MVP이므로 샘플 데이터 중심으로 구현한다.
- 웹 배포를 고려해 브라우저 환경에서도 깨지지 않는 UI/저장 동작을 우선한다.
- 웹 빌드 시 브라우저 저장소 제약(사생활 보호 모드, 쿠키/스토리지 차단 등)을 안내할 수 있어야 한다.
- 비디오 인트로는 Godot 기본 지원 포맷 기준으로 `.ogv` 사용을 전제로 한다.

## 완료 조건
- 플레이어가 새 게임을 시작하고 이름을 확정할 수 있다.
- 게임 화면에 진입해 방향 전환과 핫스팟 클릭이 가능하다.
- 샘플 대화와 선택지가 정상 동작한다.
- 호감도/스킬/도감/매뉴얼 중 최소 1개 이상이 플레이 도중 변화한다.
- 시간대가 행동 소모에 따라 전환된다.
- 3개 슬롯에 대해 저장/로드가 동작한다.
- 타이틀로 돌아온 뒤 이어하기로 기존 진행을 복원할 수 있다.
