# 괴담 도서관(도서관 사서)
# 📚 게임 프로젝트 기획서

## 🎮 프로젝트 개요

> “9개월 계약, 숙식제공, 복지무료…? 최저시급할인가에, 영생보장? 사이비 아니야? 이런 곳에 누가 취직한다고.”
> ***내가 취직했다.***
>
> 이제 살 길은 두 가지다.
> 업무를 잘 해서 정규직이 되거나… 이용자들의 눈에 들어 이직하거나.
>
> **매뉴얼 괴담 기반 포인트 앤 클릭 방 갇힘 게임**
> 신입 사서는 기묘한 손님들 사이에서 살아남을 수 있을 것인가!

---

## 1. 🎯 기본 설정

* **장르**: 방탈출형 추리 + 선택지 기반 이벤트
* **플레이 방식**: 탐색(클릭) + 선택지 + 이벤트
* **난이도**: 입문용 (공략 없이 진행 가능)
* **연령 대상**: 12~15세 (스핀오프 별도)

---

## 2. 🧭 인트로 시스템

### 최초 실행

* 허구 안내 문구 출력
* 촛불 배경 애니메이션 재생

### 이름 입력

* 텍스트 입력
* 재확인 질문
* 확정 시 변경 불가

---

## 3. 🏠 메인 메뉴

### 타이틀 화면

* 게임 타이틀
* 배경 이미지

### 메뉴 구성

* 새 게임 (초기화)
* 이어하기 (세이브 슬롯 3개)
* 근무기록 (업적 / 컬렉션)
* 설정
* 크레딧

---

## 4. 🖥️ 게임 화면 구조

### 4.1 기본 화면

* 중앙: 도서관 내부
* 이동: 좌 / 우 / 하

### 4.2 상호작용 요소

#### 중앙

* 책상 (메인 인터페이스)
* 리어카
* 천장 / 책장
* 찢긴 종이 (수집 요소)

#### 좌측

* 자동 반납기

#### 우측

* 창문 (시간/계절 변화 반영)
* 라디에이터

#### 하단

* 서랍 3칸

  * 도감 / 매뉴얼
  * 식사
  * 수집 아이템
* 침낭

  * 저장 기능

---

## 5. 🎨 UI 시스템

* 대사 출력: 책상 위 말풍선
* 선택지:

  * 최대 3개
  * 캐릭터 주변 배치

---

## 6. 🔁 핵심 게임 루프

### 6.1 시간 흐름

```
낮 → 밤 → 자유시간 → 반복
```

* 이벤트 n회 진행 시 시간 변화

---

### 6.2 이벤트 구조

#### 이벤트 타입

* 조우 (퍼즐 / 위험 회피)
* 접대 (대화 / 호감도)
* 대여 (책 빌려주기)

#### 특징

* 모든 단계가 항상 진행되지는 않음
* 이벤트 1회 = 캐릭터 1회 만남

---

## 7. 🔍 조우 시스템 (퍼즐)

* 배경 클릭 탐색
* 단서 수집
* 퍼즐 해결
* 방탈출형 구조 (씬 확대 / 전환)

### 기능

* 괴물 정체 파악
* 공격 회피

### 도감 시스템

* 반복 조우 시 정보 축적

---

## 8. 💬 접대 시스템 (호감도)

* 선택지 기반 대화
* 매뉴얼 / 도감 참고 필요

### 결과

* 호감도 상승 / 하락

---

## 9. 📖 대여 시스템

### 흐름

1. 요청 확인
2. 책장 선택
3. 자동 책 제공

### 옵션

* 미니게임 (선택 / 스킵 가능)

---

## 10. 👥 캐릭터 시스템

* 등장 방식: 랜덤 등장

### 이벤트 구조

* 캐릭터별 고정 이벤트 순서
* 만남 횟수 기반 진행

---

## 11. 📈 진행도 시스템

### 기준

* 이벤트 누적

### 변화 요소

* 스토리 진행 (떡밥)
* 맵 변화 (서랍 해금 등)
* 주인공 독백
* 계절 이벤트
* 신규 캐릭터 등장

---

## 12. 🎬 엔딩 시스템

### 1회차

* 최종 보스 이벤트 이후
* 친밀도(귀의도) 기반 엔딩 분기

### 2회차

* 플레이어 개입 루트 활성화

### 보스전

* 미니게임 방식

---

## 13. 💾 세이브 시스템

* 저장 위치: 침낭
* 슬롯: 3개

---

## 14. 🧾 수집 요소

* 도감
* 매뉴얼
* 찢긴 종이 (귀의도 관련)
* 업적

---

## 15. 🛌 기타 시스템

### 자유시간

* 독서 → 스킬 획득
* 휴식 → 시간 스킵

---

# ✅ 핵심 구조 요약 (개발 관점)

## 🔄 루프 구조

### 탐색 루프

```
이동 → 클릭 → 이벤트
```

### 진행 루프

```
이벤트 n회 → 시간 변화
```
